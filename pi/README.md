# pi

User-level configuration for [pi](https://github.com/badlogic/pi-mono), the AI coding agent.

Symlinked into `~/.pi/agent/` by `scripts/setup/setup_pi.sh`. The dotfiles repo is the single source of truth — when pi updates files under `agent/agents/` or `agent/skills/` (via `pi install` / `pi update`), the changes land in this repo and need to be committed.

## What's tracked

| Path | Symlink? | Why |
|---|---|---|
| `agent/settings.json` | yes | User-level defaults (model, packages, plugin config) |
| `agent/models.json` | yes | Model registry overrides (context windows, image input, etc.) |
| `agent/AGENTS.md` | yes | Inherits into all subagents |
| `agent/agents/*.md` (43 files) | yes — directory | Persona definitions shipped by installed packages |
| `agent/skills/*` (39 dirs) | yes — directory | Skill definitions shipped by installed packages |
| `agent/compound-engineering/install-manifest.json` | yes | Plugin-managed, but worth tracking for reproducibility |

## What stays in `~/.pi` (NOT in dotfiles)

| Path | Why exclude |
|---|---|
| `~/.pi/agent/auth.json` | Secrets (mode 600) |
| `~/.pi/agent/sessions/` | Runtime session cache, large |
| `~/.pi/agent/run-history.jsonl` | Runtime append-only log |
| `~/.pi/agent/trust.json` | Per-machine project trust decisions |
| `~/.pi/agent/npm/` | Installed package code, regenerated by `pi install` |
| `~/.pi/agent/prompts/` | Empty (pi uses `~/.pi/agent/agents/` instead) |
| `~/.pi/models/` | Managed by `pi-session-manager` extension |
| `~/.pi/pi-session-manager/` | Extension state |
| `~/.pi/agent/extensions/linear/` | Managed by `@alasano/pi-linear` package |
| `~/.pi/.git/` | The `~/.pi` repo is its own git repo (per your instruction) |

## Gotchas

1. **Last-changelog-version noise** — `settings.json` includes `lastChangelogVersion`, which pi bumps on every `pi update`. This creates a diff in the dotfiles repo across machines. Acceptable per project policy.
2. **`pi install` writes into dotfiles** — When you install/update a package, files land in `~/dotfiles/pi/agent/agents/` and `~/dotfiles/pi/agent/skills/`. Commit those changes afterward.
3. **`~/.pi` git status will show typechanged files** — Symlinks appear as typechanges in pi's own git repo. Either `git update-index --skip-worktree` the symlinked files or accept the noise.
4. **Project-level `.pi/settings.json` is not a dotfile** — Per-project config stays in the project's own git.

## Setup

The symlinks are created automatically by the top-level `setup.sh` (via `scripts/setup/setup_pi.sh`). To re-run just the pi step:

```bash
~/dotfiles/scripts/setup/setup_pi.sh
```
